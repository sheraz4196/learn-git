Git Branch and Commit Guidelines

Branch Names

Branch names should be descriptive and follow a consistent format. We use the following format:

feat/your-descriptive-branch-name

For example:

feat/add-home-page-ui-and-functionality

fix/ui-design-for-homepage

Commit Messages

Commit messages should be clear and concise, describing the changes made in the commit. Follow these guidelines:

  • Start with a tag indicating the type of change (e.g., "feat" for new features, "fix" for bug fixes).
  • Followed by a colon and a brief description of the changes.
  • Limit the line length to around 50 characters.

For example:

feat: Add homepage hero section

Example Commit

Here's an example of a well-formatted commit:

    
      feat: Add homepage UI hero section
    
    
      feat: contact us button scrolls to contact us form
    
    
      fix: issue with the image overlapping over content
    
    
      fix: issues in ui on mobile screenss
    
    
      refactor: simplified the code
    
    
      refactor: improved code quality
